## ACID — свойства транзакций 

### A — Atomicity (Атомарность)
Транзакция выполняется целиком или не выполняется вообще   

### C — Consistency (Согласованность)                              

Транзакция переводит БД из одного корректного  
состояния в другое                                         

### I — Isolation (Изоляция)                       
Параллельные транзакции не должны мешать друг другу        

### D — Durability (Долговечность)                                 
После commit данные гарантированно сохранены

##  4 уровня изоляции
## READ_UNCOMMITTED  
Что разрешено:  
Dirty Read  
Non-Repeatable Read  
Phantom Read

Что запрещено:  
Lost Update  

Как работает:  
• Читаем даже незакоммиченные данные  
• Нет блокировок на чтение  
• Максимальная производительность  
• Минимальная консистентность                       
                                                    
Когда использовать:  
• Почти никогда!  
• Только для аналитических запросов, где точность н важна  
• Когда производительность критичнее консистентности 

## READ_COMMITTED  
Что разрешено:  
Non-Repeatable Read  
Phantom Read

Как работает:  
• Читаем только закоммиченные данные  
• Блокировки только на запись (не на чтение)

Когда использовать:  
• По умолчанию для большинства приложений  
• Баланс между производительностью и консистентностью

## REPEATABLE_READ
Что разрешено:  
Phantom Read  

Как работает:  
• Все SELECT в транзакции видят ОДИНАКОВЫЙ снимок данных  
• Блокировки на чтение (shared locks)  
• Другие транзакции не могут изменить прочитанные строки 

## SERIALIZABLE  
Как работает:  
• Транзакции выполняются так, будто они последовательные
