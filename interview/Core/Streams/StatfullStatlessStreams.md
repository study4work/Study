## Stream — это ленивый конвейер операций. Он состоит из:
1. Source (источник) — коллекция, массив, генератор
2. Intermediate operations (промежуточные операции) — map, filter, sorted и т.д.
3. Terminal operation (терминальная операция) — collect, forEach, reduce, count

## Stateless операции (Без состояния)

Операция, которая обрабатывает каждый элемент независимо от других элементов. Ей не нужно помнить, что было до или после текущего элемента.

Примеры:  
map() — преобразует каждый элемент независимо  
filter() — проверяет каждый элемент независимо  
peek() — выполняет действие для каждого элемента  
flatMap() — преобразует каждый элемент в поток и "сплющивает"

```java
List<String> names = List.of("Alice", "Bob", "Charlie");

List<String> result = names.stream()
    .filter(name -> name.length() > 3)  // Stateless: проверяет каждый name независимо
    .map(String::toUpperCase)            // Stateless: преобразует каждый name независимо
    .collect(Collectors.toList());

// Результат: ["ALICE", "CHARLIE"]
```

Почему это важно: Stateless операции идеальны для параллелизма. Каждый элемент можно обрабатывать в отдельном потоке, не worrying о синхронизации.

## Stateful операции (С состоянием)

Операция, которая должна помнить информацию о других элементах потока, чтобы обработать текущий элемент. Ей нужно "состояние" (state).

Примеры:  
distinct() — должен помнить все встреченные элементы, чтобы отфильтровать дубликаты  
sorted() — должен увидеть ВСЕ элементы, чтобы отсортировать их  
limit() / skip() — должны помнить, сколько элементов уже обработано  
reduce() — накапливает результат (частично stateful)  
collect() — накапливает результат в коллекцию  

```java
List<Integer> numbers = List.of(3, 1, 4, 1, 5, 9, 2, 6, 5);

List<Integer> result = numbers.stream()
    .distinct()  // Stateful: должен помнить все встреченные числа
    .sorted()    // Stateful: должен увидеть ВСЕ числа, чтобы отсортировать
    .limit(3)    // Stateful: должен помнить, сколько элементов уже взял
    .collect(Collectors.toList());

// Результат: [1, 2, 3]
```

Почему это важно: Stateful операции проблематичны для параллелизма. Они требуют синхронизации или буферизации, что убивает производительность.

## Ленивая инициализация (Lazy Evaluation)

Ленивая инициализация — это когда вычисления откладываются до тех пор, пока результат действительно не понадобится.

В Stream API это означает: промежуточные операции не выполняются, пока не вызвана терминальная операция.
