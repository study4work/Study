## Функциональный интерфейс 
— это интерфейс, у которого ровно один абстрактный метод. Это ключевое отличие. Он представляет собой контракт для одного действия, что позволяет использовать его с лямбда-выражениями.

Правила функционального интерфейса:  
1 Ровно один абстрактный метод  
2 Любое количество default методов (с реализацией)  
3 Любое количество static методов (с реализацией)  
4 Методы из Object не считаются (equals, hashCode, toString)  

Важно: если вы явно объявите метод с сигнатурой как у Object, это не сломает функциональность:  
```java
public interface MyInterface {
    void doWork();
    
    // Это НЕ добавляет абстрактный метод, т.к. такая сигнатура есть в Object
    boolean equals(Object obj);  // ✅ OK, остаётся функциональным
}
```

## Аннотация @FunctionalInterface
✅ Сообщает компилятору: "этот интерфейс должен быть функциональным"  
✅ Компилятор проверит и выдаст ошибку, если абстрактных методов ≠ 1  
❌ НЕ обязательна — интерфейс всё равно будет функциональным без неё  

Связь с лямбда-выражениями

Лямбда — это краткая запись реализации функционального интерфейса.

Без лямбды (анонимный класс):
```java
Runnable task = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello!");
    }
};
```

С лямбдой:
```java
Runnable task = () -> System.out.println("Hello!");
```
Как компилятор понимает, что это Runnable?  
По контексту (target typing). Лямбда всегда должна быть присвоена функциональному интерфейсу. Невозможно использовать лямбду без контекста.
