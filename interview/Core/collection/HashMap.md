### HashMap — это массив бакетов, где каждый бакет — это либо null, либо голова связного списка (Node), либо корень красно-чёрного дерева (TreeNode). Реализует интерфейс Map и наследуется от AbstractMap.

### AbstractMap — это базовый класс, который реализует интерфейс Map через минимальный набор абстрактных методов. Наследнику нужно реализовать только entrySet(), а всё остальное (get, put, remove, size и т.д.) уже готово.

![img.png](/src/main/resources/images/hasMap.png)

```java
public class HashMap<K,V> extends AbstractMap<K,V> {
    
    // Таблица бакетов — массив "корзин"
    transient Node<K,V>[] table;
    
    // Множество записей (для entrySet())
    transient Set<Map.Entry<K,V>> entrySet;
    
    // Текущее количество элементов
    transient int size;
    
    // Порог для resize: capacity * loadFactor
    int threshold;
    
    // Коэффициент загрузки (по умолчанию 0.75)
    final float loadFactor;
}
```

## Структура связанного списка

```java
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;      // хэш ключа (кэшируется!)
    final K key;         // ключ (final — нельзя изменить)
    V value;             // значение
    Node<K,V> next;      // следующий узел в цепочке
}
```

## Когда LinkedList превращается в Red-Black Tree
Treeification происходит только при одновременном выполнении ДВУХ условий:
```
┌──────────────────────────────────────────────────────────────┐
│  УСЛОВИЯ TREEIFICATION                                       │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  ✅ 1. Длина цепочки в бакете >= 8 (TREEIFY_THRESHOLD)      │
│          И                                                    │
│  ✅ 2. capacity таблицы >= 64 (MIN_TREEIFY_CAPACITY)        │
│                                                               │
│  Если условие 2 не выполнено → вместо treeify будет RESIZE  │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

## Три сценария при коллизии
```
┌─────────────────────────────────────────────────────────────────┐
│  СЦЕНАРИЙ 1: Бакет достиг 8 узлов, capacity < 64              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  capacity = 32, в бакете 8 узлов                               │
│                                                                  │
│  ┌──────┐    ┌──────┐    ┌──────┐              ┌──────┐       │
│  │Node 0│───▶│Node 1│───▶│Node 2│───▶ ... ────▶│Node 7│       │
│  └──────┘    └──────┘    └──────┘              └──────┘       │
│                                                                  │
│  Действие: RESIZE (capacity → 64)                              │
│  Результат: цепочки разбиваются, коллизии уменьшаются          │
│  Дерево НЕ строится!                                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  СЦЕНАРИЙ 2: Бакет достиг 8 узлов, capacity >= 64             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  capacity = 64, в бакете 8 узлов                               │
│                                                                  │
│  Действие: TREEIFY (превращаем в RB-дерево)                    │
│                                                                  │
│  До:                          После:                            │
│  N0→N1→N2→...→N7                  N4                           │
│                                  /  \                           │
│                                N2    N6                         │
│                               / \   / \                         │
│                              N1 N3 N5  N7                       │
│                                                                  │
│  Поиск: O(n) → O(log n)                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  СЦЕНАРИЙ 3: Бакет достиг 8 узлов, capacity = 16              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  capacity = 16, в бакете 8 узлов                               │
│                                                                  │
│  Действие: RESIZE                                               │
│    • capacity: 16 → 32                                          │
│    • threshold: 12 → 24                                         │
│    • все элементы перераспределяются                            │
│                                                                  │
│  После resize:                                                  │
│    • в бакете осталось < 8 узлов → treeify не нужен            │
│    • ИЛИ всё ещё >= 8 → снова проверяем capacity               │
│    • Если capacity всё ещё < 64 → снова resize                 │
│    • Пока не достигнем 64, treeify не произойдёт!              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Обратный процесс: UNTREEIFY
```
┌──────────────────────────────────────────────────────────────┐
│  ДЕГРАДАЦИЯ ДЕРЕВА ОБРАТНО В LIST                            │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  Происходит при:                                             │
│  • remove() уменьшил размер дерева до <= 6                  │
│    (UNTREEIFY_THRESHOLD = 6)                                 │
│  • resize() разбил элементы по разным бакетам               │
│                                                               │
│  Почему 6, а не 8?                                           │
│  → Hysteresis (защита от частых переключений)               │
│  → Если бы было 8, то при size=8 дерево → list,             │
│     потом добавили элемент → list → tree, и так бесконечно  │
│                                                               │
│  Дерево (7 узлов)    →    После remove() (6 узлов)          │
│       N4                    N0→N1→N2→N3→N4→N5→N6            │
│      /  \                                                  │
│    N2    N6                                                │
│   / \   /                                                  │
│  N1  N3 N5                                                 │
│  │                                                         │
│  N0                                                        │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```
